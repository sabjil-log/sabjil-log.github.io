❯ 말싸움 끝내는 법 — tcpdump로 '누가 응답을 안 하는지' 눈으로 보기

한 줄 요약: "방화벽은 열려 있어요" vs "서버는 정상이에요"의 말싸움은 tcpdump 한 줄이 끝냅니다. 패킷이 도착했는지, 답이 나갔는지를 눈으로 보면 책임 구간이 물리적으로 갈려요. 필요한 건 필터 문법 조금과 SYN 3종 시나리오 읽는 법뿐입니다.

언제 꺼내는 칼인가

계층 진단(dig→ping→nc→ss)을 다 돌았는데도 애매할 때, 또는 두 팀이 서로 "우리 쪽은 정상"이라고 우길 때. tcpdump는 네트워크의 CCTV입니다 — 추측이 아니라 녹화 영상으로 말하게 해줘요.

최소 문법 — 이것만 외우면 된다

sudo tcpdump -ni any port 8080
#        │ │  │    └ 필터: 8080 포트만
#        │ │  └ any: 모든 인터페이스에서
#        │ └ n: 이름 해석 끄기 (느려지지 않게, 숫자 그대로)
#        └ i: 인터페이스 지정

필터는 영어처럼 조합합니다.

sudo tcpdump -ni any host 10.0.3.15                  # 이 상대와의 대화만
sudo tcpdump -ni any host 10.0.3.15 and port 5432    # 그 상대의 DB 통신만
sudo tcpdump -ni any src 10.0.1.0/24 and dst port 443 # 이 대역에서 443으로 오는 것
sudo tcpdump -ni any port 8080 -c 20                  # 20개만 잡고 자동 종료

-c 를 습관처럼 붙이세요 — 바쁜 서버에서 필터 없이 돌리면 화면이 폭포가 됩니다.

핵심: SYN 3종 시나리오

TCP 연결은 노크(SYN) → 응답(SYN-ACK) → 확인(ACK)으로 시작합니다. 접속 문제는 이 첫 악수에서 대부분 판가름 나요. 서버 쪽에서 캡처를 켜고 클라이언트가 접속을 시도하게 한 뒤, 세 가지 패턴 중 뭐가 보이는지만 읽으면 됩니다.

패턴 ① — 아무것도 안 보인다

(정적)

노크 자체가 도착 안 함 → 내 서버 앞 어딘가(방화벽, NACL, 라우팅, 보안그룹)가 범인. 서버와 앱은 무죄입니다. 네트워크 팀에 "SYN이 서버까지 안 옵니다"라고 증거로 말할 수 있어요.

패턴 ② — SYN은 오는데 서버가 RST로 답한다

10.0.1.5.54321 > 10.0.3.15.8080: Flags [S], ...     ← 노크 도착
10.0.3.15.8080 > 10.0.1.5.54321: Flags [R.], ...    ← 서버: "그런 포트 없는데?"

패킷은 도착했는데 그 포트에 듣는 프로세스가 없다는 뜻 → 앱이 안 떠 있거나 다른 포트/127.0.0.1 바인딩. ss -tlnp 로 확인하세요. 네트워크는 무죄.

패턴 ③ — SYN이 반복되는데 응답이 없다

Flags [S] ... → (응답 없음) → Flags [S] 재전송 → Flags [S] 재전송...

노크는 오는데 답이 안 나감 → 로컬 방화벽(iptables)이 조용히 버리거나(DROP), 응답이 나가는 길이 막힌 것 — 스테이트리스 NACL의 아웃바운드 임시 포트, 비대칭 라우팅이 단골. 헬스체크 글의 그 함정이 화면에 잡히는 순간입니다.

내용까지 봐야 할 때

sudo tcpdump -ni any port 80 -A | grep -E "GET|POST|Host"   # HTTP 평문 엿보기
sudo tcpdump -ni any port 5432 -w db.pcap                   # 파일로 저장
# → db.pcap 을 내려받아 Wireshark로 여는 게 눈이 편합니다

주의 둘 — HTTPS는 내용이 암호화라 -A로 봐도 의미가 없고(악수 과정까지만 보임), 캡처 파일에는 평문 통신의 비밀번호·토큰이 그대로 담깁니다. pcap 파일을 슬랙에 아무렇게나 던지지 마세요. 그 자체가 유출입니다.

실전 루틴

# 터미널 1 (서버): 녹화 시작
sudo tcpdump -ni any port 8080 -c 30

# 터미널 2 (클라이언트): 재현
curl -m 5 http://server:8080/health

재현 트래픽을 내가 만들면서 캡처하는 것 — 이게 요령입니다. 흘러가는 패킷 바다에서 낚는 게 아니라, 내가 던진 돌의 파문만 보는 거죠.

한 줄 정리

tcpdump는 필터 3개(host/port/src·dst) + SYN 3패턴이면 실무의 8할입니다. 정적이면 앞단 방화벽, RST면 앱 부재, SYN 반복이면 응답 경로 — 추측을 증거로 바꾸는 30초예요.
