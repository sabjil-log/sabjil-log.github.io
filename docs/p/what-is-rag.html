❯ RAG가 뭐길래 — LLM한테 '오픈북 시험'을 보게 하는 법

한 줄 요약: LLM은 학습 때 외운 것만 답하는 클로즈드북 시험 상태입니다. RAG는 질문에 답하기 직전에 관련 문서를 찾아 책상 위에 올려주는 오픈북 시험으로 바꿔주는 기법이에요.

왜 필요한가

아무리 똑똑한 LLM도 두 가지를 못 합니다. 사내 문서처럼 학습한 적 없는 내용, 그리고 최신 정보. 모르는 걸 물으면 그럴듯하게 지어내죠(환각). 그렇다고 회사 문서로 매번 모델을 새로 학습(파인튜닝)시키는 건 비싸고 느립니다.

비유: 클로즈드북 vs 오픈북

시험을 떠올려 보세요. 클로즈드북은 머리에 외운 것만 씁니다. 오픈북은 책을 뒤져 근거를 보고 답하죠. RAG는 여기서 한 발 더 나갑니다 — 시험 직전에 조교가 관련 페이지만 쏙 뽑아 책상에 올려주는 것. 학생(LLM)은 그 페이지를 근거로 답만 잘 쓰면 됩니다.

어떻게 "관련 페이지"를 찾나 — 4단계

  1. 자르기(chunk): 문서를 문단 단위로 잘게 나눕니다.
  2. 임베딩(embedding): 각 조각을 숫자 벡터로 바꿉니다. 핵심은 뜻이 비슷하면 벡터도 가깝다는 것. "강아지"와 "반려견"은 지도에서 이웃, "환율"은 저 멀리.
  3. 검색: 질문도 벡터로 바꿔서, 가장 가까운 조각 몇 개(top-k)를 벡터 DB에서 꺼냅니다.
  4. 답하기: 꺼낸 조각을 질문과 함께 프롬프트에 넣어, LLM이 그 근거만 보고 답하게 합니다.

개념 코드 스케치

실제 구현이 아니라 흐름만 보여주는 뼈대입니다.

# 1) 문서를 잘라 임베딩해서 저장
chunks = split(document)
index = [(embed(c), c) for c in chunks]      # (벡터, 원문) 쌍

# 2) 질문과 가장 가까운 조각 top-k 검색
q = embed(question)
top = nearest(index, q, k=4)                 # 벡터 거리로 정렬해 상위 4개

# 3) 근거를 붙여 LLM에 질문
prompt = f"""아래 근거만 사용해서 답해. 없으면 모른다고 해.

[근거]
{top}

[질문]
{question}"""
answer = llm(prompt)

흔한 함정

  • 검색이 엉뚱하면 답도 엉뚱합니다(garbage in, garbage out). 그래서 뽑은 후보를 리랭커(reranker)로 한 번 더 정렬해, 진짜 관련 있는 조각을 위로 올립니다. 1차 검색은 빠르지만 거칠고, 리랭커는 느리지만 정확 — 둘을 조합해요.
  • 자르는 크기가 품질을 좌우합니다. 너무 크게 자르면 한 조각에 잡음이 섞이고, 너무 잘게 자르면 문맥이 끊깁니다.
  • "근거에 없으면 모른다고 해"를 프롬프트에 명시하지 않으면, 모델은 또 지어냅니다.

한 줄 정리

RAG는 모델을 더 똑똑하게 만드는 게 아니라, 답하기 직전에 정답 페이지를 쥐여주는 겁니다. 그래서 승부는 모델보다 "얼마나 관련 있는 조각을 잘 찾아 오느냐"에서 갈려요.
