❯ grep·awk·sed 삼총사 — 로그에서 원하는 값만 뽑아내는 법

한 줄 요약: 역할만 외우면 됩니다. grep = 줄 고르기, awk = 칸 자르기, sed = 글자 바꾸기. 이 셋을 파이프(|)로 이으면 수만 줄 로그에서 원하는 값만 한 줄로 뽑아냅니다.

이런 상황

접속 로그 수십만 줄에서 "500 에러가 난 요청의 IP만, 많이 나온 순으로" 보고 싶습니다. 로그를 열어 눈으로 찾는 건 불가능하고, 엑셀로 옮기기엔 너무 큽니다. 삼총사가 나설 차례예요.

비유: 조리 라인

주방 조리 라인을 떠올리면 정확합니다.

  • grep — 재료 고르는 사람. 조건에 맞는 줄만 통과시킵니다.
  • awk — 손질하는 사람. 통과한 줄을 칸(필드) 으로 잘라 필요한 칸만 꺼냅니다.
  • sed — 다듬는 사람. 꺼낸 값의 글자를 바꾸거나 지웁니다.

파이프(|)는 접시를 다음 사람에게 넘기는 컨베이어고요.

하나씩 최소한만

grep — 이 줄 줘

grep " 500 " access.log            # 500 이 포함된 줄
grep -v "health" access.log        # health 는 빼고 (-v = 반전)
grep -c " 500 " access.log         # 몇 줄인지 개수만
grep -E "50[023]" access.log       # 정규식: 500/502/503

awk — 이 칸 줘 (기본 구분자는 공백)

awk '{print $1}' access.log        # 1번째 칸 (보통 IP)
awk '{print $1, $9}' access.log    # 1번째와 9번째 칸
awk '$9 == 500 {print $1}' access.log   # 9번 칸이 500인 줄의 IP만 — grep 없이도 조건 가능

sed — 이 글자 바꿔 줘

sed 's/오류/ERROR/' app.log        # 각 줄 첫 '오류' → ERROR
sed 's/오류/ERROR/g' app.log       # 줄 안 전부 (g)
sed 's/"//g' data.txt              # 따옴표 제거

조합 — 실전 한 줄

"500 에러 낸 IP를 많이 나온 순으로 톱10":

grep " 500 " access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head
#      ↑ 500 줄만          ↑ IP만        ↑ 정렬  ↑ 중복 세기  ↑ 많은 순   ↑ 10개

sort | uniq -c | sort -rn 은 "세어서 랭킹 만들기" 관용구입니다. 통째로 외워두면 평생 씁니다.

하나 더 — "에러 시각과 메시지만, 따옴표 떼고":

grep ERROR app.log | awk '{print $1, $2, $NF}' | sed 's/"//g'
# $NF = 마지막 칸

흔한 함정

  • awk 조건에서 == 는 숫자/문자 비교, 패턴 매칭은 ~ 입니다: awk '$9 ~ /^5/ ...' (5xx 전부).
  • sed 의 s/a/b/ 는 기본이 줄당 한 번. 전부 바꾸려면 g 를 잊지 마세요.
  • 구분자가 공백이 아니면 awk 에 -F 를 줍니다: awk -F',' '{print $3}' (CSV의 3번째 칸).

한 줄 정리

grep으로 줄을 고르고 → awk로 칸을 꺼내고 → sed로 글자를 다듬는다. 여기에 sort | uniq -c | sort -rn 랭킹 관용구까지 — 이 네 조각이면 로그 분석의 8할이 끝납니다.
