❯ ACG(보안그룹) vs NACL 총정리 — 스테이트풀과 스테이트리스, 한 번에 끝내기

한 줄 요약: 인스턴스 방화벽(ACG/보안그룹)은 대화를 기억하는 경비(스테이트풀) — 나간 요청의 응답은 자동 통과. 서브넷 방화벽(NACL)은 기억이 없는 경비(스테이트리스) — 응답도 규칙이 있어야 나갑니다. 실전 공식은 하나: 주 방어는 ACG로, NACL은 넓게 열어두고 서브넷 차원의 큰 차단에만.

두 경비의 위치부터

건물 비유로 —

인터넷 → [NACL: 층 입구 경비] → 서브넷(층) → [ACG: 각 방문 앞 경비] → 서버(방)
  • NACL = 층 전체 입구의 경비. 그 층의 모든 방에 일괄 적용.
  • ACG/보안그룹 = 방문마다 붙은 경비. 그 방(인스턴스)에만 적용.

패킷은 둘 다 통과해야 합니다. 그래서 "방화벽 열었는데요?"라는 말은 항상 "둘 중 어느 걸요?"를 되물어야 해요.

결정적 차이: 기억력

ACG는 스테이트풀 — 대화를 기억합니다. 방 안에서 "피자 주문 나갑니다" 하고 나간 걸 봤으면, 배달원이 돌아올 때 규칙을 안 따져도 들여보내줍니다. 그래서 ACG는 인바운드 규칙만 고민하면 되고, 아웃바운드는 대개 전체 허용으로 둡니다.

NACL은 스테이트리스 — 기억이 없습니다. 나가는 배달 주문을 봤어도 잊어버려요. 돌아오는 배달원도 명시적 허용 규칙이 있어야 통과합니다. 여기서 그 유명한 함정이 나옵니다 —

그 함정, 정면으로: 임시 포트

클라이언트가 서버 443에 접속하면, 응답은 클라이언트의 임시 포트(1024~65535 사이 무작위)로 돌아갑니다.

요청:  클라이언트:53211 → 서버:443     (인바운드 443 허용 — 다들 함)
응답:  서버:443 → 클라이언트:53211    (아웃바운드 '임시 포트' 허용 — 다들 까먹음)

NACL에서 인바운드 443만 열고 아웃바운드를 안 열면(또는 443만 열면), 요청은 들어오는데 응답이 층 밖으로 못 나갑니다. 증상은 "연결이 되다 마는" 기묘한 행 — 헬스체크가 간헐 실패하고(그 3대장 글), tcpdump엔 SYN만 반복되고(그 패턴③), NAT 뒤에서는 더 꼬입니다. 그래서 NACL을 만질 땐 반드시 세트로:

인바운드:  서비스 포트 (443 등) 허용
아웃바운드: 임시 포트 1024-65535 허용   ← 응답이 나가는 길

차이 요약표

ACG / 보안그룹 NACL
적용 대상 인스턴스(방) 서브넷(층) 전체
상태 스테이트풀 (응답 자동) 스테이트리스 (응답도 규칙 필요)
규칙 종류 허용만 허용 + 거부 가능
평가 방식 전체 규칙 종합 번호 순서대로, 첫 매치에서 종료
기본 성향 전부 거부에서 시작 기본 NACL은 전부 허용

두 가지가 ACG엔 없고 NACL에만 있습니다 — 명시적 거부와 규칙 순서. "이 IP 대역은 무조건 차단"은 ACG로는 못 하고(허용만 가능) NACL의 거부 규칙이 필요해요. 대신 순서가 생기니 사고도 생깁니다: 낮은 번호의 넓은 허용이 높은 번호의 거부보다 먼저 매치되면 거부가 무력화되는 식. NACL 규칙이 많아질수록 순서 퍼즐이 되는 이유입니다.

실전 설계 공식

복잡해 보이지만 업계 정석은 단순합니다.

  1. 주 방어선은 ACG. 스테이트풀이라 실수 여지가 적고, 인스턴스 단위라 정밀합니다. "웹 ACG는 LB에서 오는 8080만, DB ACG는 웹 ACG로부터 5432만" — ACG가 ACG를 참조하는 체이닝이 백미예요(IP가 아니라 역할로 여는 것).
  2. NACL은 기본(전부 허용)에 가깝게 유지. 굳이 조이지 않습니다 — 조일수록 임시 포트·순서 함정의 표면적만 늘어요.
  3. NACL을 쓰는 순간은 딱 두 가지. 특정 대역의 명시적 차단(공격 IP, 정책상 금지 대역)과, 서브넷 계층 자체의 거친 분리(예: DB 층에는 웹 층 대역 외 접근 금지라는 굵은 선). 정밀함은 ACG에게.

이러면 "이중 방화벽"이 퍼즐이 아니라 역할 분담이 됩니다 — 층 경비는 굵은 선만, 방 경비가 정밀 심사.

디버깅 순서

연결이 안 되면: ① ACG 인바운드에 출발지·포트가 있나 → ② 그 서브넷의 NACL 인바운드에 있나 → ③ NACL 아웃바운드에 임시 포트가 있나 → ④ 그래도면 tcpdump로 패킷의 행방을(그 글의 3패턴). 경험상 ③이 범인인 비율이 부끄러울 만큼 높습니다.

한 줄 정리

ACG는 기억하는 경비, NACL은 규칙만 보는 경비. 주 방어는 ACG(+ACG 체이닝), NACL은 넓게 두고 명시적 차단·층 분리에만 — 그리고 NACL을 조였다면 아웃바운드 임시 포트부터 확인하세요. 이 글 하나로 그 함정은 졸업입니다.
