❯ 손으로 돌리면 되는데 크론으로는 안 되는 스크립트 — 환경변수의 배신

한 줄 요약: 크론은 스크립트를 당신의 터미널이 아니라, 텅 빈 환경에서 실행합니다. PATH는 최소한이고, .bashrc도 안 읽고, 작업 디렉토리도 다릅니다. "손으로는 되는데 크론으로는 안 되는" 미스터리의 답은 거의 항상 여기에 있어요.

증상

백업 스크립트를 만들어 터미널에서 실행 — 완벽하게 됩니다. crontab에 등록하고 퇴근. 다음 날 보니 백업이 없습니다. 크론 자체는 돌았는데(로그에 흔적 있음) 스크립트만 실패했어요. 코드를 아무리 봐도 문제가 없습니다. 당연하죠 — 코드 문제가 아니니까.

비유: 내 주방 vs 빈 주방

터미널에서 실행하는 건 내 주방에서 요리하는 것입니다. 조미료(환경변수), 도구 위치(PATH), 익숙한 배치(작업 디렉토리)가 다 세팅돼 있죠. 크론은 같은 레시피를 아무것도 없는 빈 주방에서 실행합니다. "소금 어디 있지?"에서 요리가 멈추는 겁니다.

용의자 ① — 빈약한 PATH

터미널의 PATH에는 /usr/local/bin 등 온갖 경로가 들어 있지만, 크론의 PATH는 보통 /usr/bin:/bin 이 전부입니다. 그래서 aws, kubectl, node, pyenv로 깐 python 같은 것들이 "command not found" 로 죽어요.

빈 주방을 직접 재현해보면 바로 이해됩니다.

# 크론과 비슷한 '빈 환경'에서 내 스크립트를 실행해보기
env -i /bin/sh -c '/home/me/backup.sh'
# 터미널에선 되는데 여기서 죽으면 → 환경 의존성 확진

해법은 둘 중 하나 — crontab 상단에 PATH를 명시하거나, 스크립트 안에서 절대경로를 씁니다.

# crontab -e
PATH=/usr/local/bin:/usr/bin:/bin
0 3 * * * /home/me/backup.sh >> /var/log/backup.log 2>&1

# 또는 스크립트에서 절대경로로
/usr/local/bin/aws s3 sync /data s3://my-backup/

용의자 ② — .bashrc는 안 읽힌다

export API_KEY=... 를 .bashrc에 넣어뒀다면, 크론에선 없는 값입니다. 크론은 로그인 셸이 아니라서 그런 파일들을 읽지 않아요. 스크립트가 조용히 빈 값으로 돌다가 이상하게 실패하는 패턴이죠.

해법: 필요한 변수를 crontab에 직접 적거나, 스크립트 첫머리에서 env 파일을 명시적으로 로드합니다.

#!/bin/bash
set -a; source /home/me/.env; set +a   # .env의 변수를 export하며 로드

(비밀값이라면 파일 권한 600, 더 나아가면 Secret Manager에서 실행 시점에 받아오는 게 상책입니다.)

용의자 ③ — 작업 디렉토리와 상대경로

터미널에선 스크립트 폴더에서 실행했겠지만, 크론의 작업 디렉토리는 홈 디렉토리입니다. ./config.json, ../data/ 같은 상대경로가 전부 엉뚱한 곳을 가리켜요.

#!/bin/bash
cd "$(dirname "$0")"     # 스크립트가 있는 곳으로 이동하고 시작 — 만능 한 줄

그 외 잔펀치들

  • 셸이 다름: 크론 기본 셸은 sh(시스템에 따라 dash)입니다. 스크립트가 bash 문법(배열, [[ ]])을 쓰면 깨져요. 셔뱅을 #!/bin/bash로 박고 실행 권한을 확인하세요.
  • % 문자: crontab 명령줄에서 %는 개행으로 해석됩니다. date +\%Y\%m\%d 처럼 이스케이프 필수 — 백업 파일명에 날짜 넣다가 자주 당합니다.
  • 출력을 버리지 마세요: 실패 원인은 stderr에 있습니다. 위 예시처럼 >> 로그파일 2>&1 를 붙여야 다음 날 아침에 볼 유언이 남아요.

진단 루틴

  1. grep CRON /var/log/syslog (또는 journalctl -u cron) — 크론이 실행 시도는 했는지
  2. 명령 끝에 >> /tmp/x.log 2>&1 붙이고 다음 주기 대기 — 유언 확보
  3. env -i /bin/sh -c '스크립트' — 빈 주방 재현으로 환경 의존성 색출

한 줄 정리

크론은 코드를 바꾸지 않습니다 — 주방을 바꿉니다. PATH 명시(또는 절대경로), env 명시적 로드, cd "$(dirname "$0")", 로그 리다이렉트. 이 네 줄을 습관으로 만들면 "크론에서만 죽는" 스크립트는 사라져요.
