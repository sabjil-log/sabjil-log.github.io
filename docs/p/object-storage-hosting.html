❯ 서버 없이 웹사이트 띄우기 2탄 — 오브젝트 스토리지 정적 호스팅

한 줄 요약: 오브젝트 스토리지는 파일 창고지만, "이 버킷의 파일을 웹으로 서빙해줘" 스위치를 켜면 웹서버가 됩니다. 서버 관리 0, 용량 걱정 0 — 대신 공개 범위, index 문서, HTTPS 세 가지 함정만 알고 시작하면 돼요.

언제 쓰나

GitHub Pages로 블로그를 띄우는 것과 같은 원리인데, 무대가 회사 클라우드입니다. 사내 대시보드, 제품 랜딩페이지, SPA(React 빌드 결과물), 문서 사이트 — HTML/JS/CSS만으로 되는 것들은 전부 후보예요. 특히 회사 인프라 안에서, 도메인·권한을 우리가 통제해야 할 때 GitHub Pages 대신 이걸 씁니다.

원리: 창고에 진열대 스위치가 있다

오브젝트 스토리지는 원래 창고입니다. 물건(파일)을 넣고 키(경로)로 꺼내는 곳이죠. 그런데 벤더들이 스위치를 하나 달아놨어요 — 정적 웹 호스팅 옵션. 켜면 창고가 진열대가 됩니다.

  • index.html 을 대문으로 지정
  • 버킷 전용 웹 주소 발급 (예: 버킷명.s3-website-리전.amazonaws.com, NCP도 유사한 웹사이트 엔드포인트 제공)
  • 이후 파일을 올리면 그대로 웹에 뜸

배포는 곧 파일 동기화입니다.

# AWS S3
aws s3 sync ./dist s3://my-site --delete

# NCP Object Storage (S3 호환 — endpoint만 지정)
aws --endpoint-url https://kr.object.ncloudstorage.com \
    s3 sync ./dist s3://my-site --delete

--delete 는 로컬에서 지운 파일을 버킷에서도 지워 빌드 결과물과 진열대를 일치시킵니다. (NCP는 boto3/CLI 버전에 따른 체크섬 호환 이슈가 있으니, 막히면 예전에 다룬 checksum 설정 글을 참고하세요.)

함정 ① — 공개 범위: 전시장이지 금고가 아니다

웹 호스팅을 켜면 버킷(의 해당 파일들)을 공개 읽기로 열게 됩니다. 여기서 사고가 나요 — 사이트 파일 옆에 백업이나 내부 문서를 같은 버킷에 두는 것.

원칙은 하나입니다. 웹 호스팅 버킷 = 100% 공개돼도 되는 파일만. 데이터용 버킷과 물리적으로 분리하세요. "버킷 안에서 폴더로 나누면 되겠지"는 정책 실수 한 번이면 무너집니다.

# 지금 뭐가 공개돼 있는지 점검 (전체 목록을 눈으로)
aws s3 ls s3://my-site --recursive | head -50

함정 ② — SPA의 404

React/Vue 라우터를 쓰는 SPA를 올리면, /about 새로고침에서 404가 납니다. 스토리지 입장에선 about이라는 파일이 없으니까요. 웹 호스팅 설정의 에러 문서를 index.html로 지정하면, 없는 경로도 index로 떨어져 라우터가 이어받습니다. SPA 호스팅의 국룰 설정입니다.

함정 ③ — HTTPS는 CDN이 담당

버킷의 웹사이트 엔드포인트는 보통 HTTP만 됩니다(커스텀 도메인 + HTTPS 불가). 요즘 브라우저 기준으론 사실상 미완성이죠. 그래서 실전 구성은 항상 이 3단입니다.

사용자 → CDN (HTTPS 인증서, 커스텀 도메인, 캐시) → 버킷 (원본)

AWS면 CloudFront + ACM 인증서, NCP면 CDN+ 에 인증서 붙이는 구성. CDN이 앞에 서는 순간 지난 글에서 다룬 캐시 갱신 문제(TTL·무효화·해시 파일명)가 그대로 따라온다는 것도 기억하세요 — 배포했는데 옛 화면이 나오면 그 글의 체크리스트대로 가면 됩니다.

비용 감각

정적 호스팅의 청구서는 저장량 + 전송량입니다. 저장은 몇 GB 수준이라 껌값이고, 변수는 전송량 — 그런데 CDN을 앞에 세우면 대부분의 트래픽이 엣지 캐시에서 소화돼 원본 전송량이 뚝 떨어집니다. 성능 때문만이 아니라 비용 때문에도 CDN이 정답인 구조예요.

한 줄 정리

버킷의 웹 호스팅 스위치 + s3 sync 배포, 여기에 전용 버킷 분리(공개 범위) / 에러 문서 = index.html (SPA) / CDN으로 HTTPS 세 가지만 챙기면, 서버 한 대 없이 운영되는 사이트가 완성됩니다. 창고를 진열대로 — 스위치는 이미 달려 있어요.
