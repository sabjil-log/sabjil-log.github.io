❯ 16GB 그래픽카드에 14B 모델이 들어가는 마법 — 4-bit 양자화

한 줄 요약: 모델 무게의 정체는 파라미터 개수 × 숫자 하나의 크기입니다. 양자화는 숫자 하나를 16비트에서 4비트로 줄여서(반올림해서) 저장하는 것 — 크기가 1/4로 줄어 14B 모델이 16GB 카드에 들어갑니다. 대가는 약간의 정밀도예요.

계산부터 해보자

모델 용량은 곱셈 하나로 감이 옵니다.

14B 모델의 무게:
FP16 (16bit = 2byte):  14B × 2byte = 28 GB   → 16GB 카드에 안 들어감
INT4 ( 4bit = 0.5byte): 14B × 0.5byte = 7 GB  → 여유 있게 들어감

여기에 추론 중 대화 기억용 캐시(KV cache)와 오버헤드가 얹히지만, 가중치가 7GB면 16GB 카드에서 컨텍스트까지 넉넉히 돌릴 수 있습니다. 모델을 줄인 게 아니라 숫자 하나하나를 짧게 적은 것 — 이게 양자화입니다.

비유: 소수점 반올림

파라미터는 0.73412885... 같은 실수 수십억 개입니다. 양자화는 이걸 "0.73쯤"으로 반올림해 저장하는 겁니다. 지도의 축척과도 같아요 — 1:5,000 지도를 1:50,000으로 줄이면 골목길 몇 개는 뭉개지지만, 서울에서 부산 가는 데는 아무 지장이 없죠. LLM의 파라미터도 상당한 중복과 여유가 있어서, 웬만한 반올림엔 성능이 크게 안 무너집니다.

다만 모든 골목이 똑같이 중요하진 않습니다. 어떤 파라미터는 뭉개지면 티가 확 나요. 그래서 좋은 양자화 기법(AWQ, GPTQ 등)은 실제 데이터를 흘려보며 중요한 가중치를 골라 정밀도를 지켜주고 나머지를 과감히 줄입니다. 무지성 반올림과 결과 차이가 꽤 큽니다.

얼마나 잃는가

  • 8-bit: 손실이 거의 체감 안 되는 수준. "고민되면 8비트"가 안전한 선택.
  • 4-bit: 대부분의 대화·요약·코딩 작업에서 준수합니다. 다만 수학·논리 추론처럼 정밀한 작업에서 미세한 열화가 먼저 드러나요.
  • 3-bit 이하: 열화가 눈에 띄기 시작. 특수한 상황 아니면 비추.

실무 감각으로는 "같은 VRAM이면, 작은 모델 FP16보다 큰 모델 4-bit가 보통 더 낫다"는 경험칙이 있습니다. 7B를 FP16으로 돌리는 것보다 14B를 4-bit로 돌리는 쪽이 체감 품질이 좋은 경우가 많아요.

이름들 정리 — AWQ, GPTQ, GGUF

모델 파일명에 붙는 꼬리표들의 정체:

  • AWQ / GPTQ: 4-bit 양자화 기법 이름. 중요 가중치를 보호하는 방식이 다름. GPU 서빙(vLLM 등)에서 주로 사용.
  • GGUF: llama.cpp 진영의 파일 포맷. Q4_K_M 같은 접미사가 양자화 수준(4비트 계열 중 중간 품질)을 표시. CPU/맥북 로컬 실행의 표준.
  • bitsandbytes: 로드 시점에 즉석 양자화하는 라이브러리(load_in_4bit=True). 파일은 원본 그대로 두고 메모리에서만 줄이는 방식.

내 카드에 들어갈지 미리 계산

def vram_estimate(params_b, bits, overhead_gb=2.0):
    """대략적 가중치 VRAM (GB). KV cache는 컨텍스트 길이에 따라 별도 증가."""
    weights = params_b * bits / 8          # B 파라미터 × byte
    return round(weights + overhead_gb, 1)

for bits in (16, 8, 4):
    print(f"14B @ {bits}bit ≈ {vram_estimate(14, bits)} GB")
# 14B @ 16bit ≈ 30.0 GB  → 16GB 불가
# 14B @  8bit ≈ 16.0 GB  → 아슬아슬 (컨텍스트 못 늘림)
# 14B @  4bit ≈  9.0 GB  → 여유. 긴 컨텍스트도 가능

실행 확인은 로드 후 한 줄이면 됩니다.

nvidia-smi --query-gpu=memory.used,memory.total --format=csv

한 줄 정리

양자화는 숫자 하나의 자릿수를 줄이는 반올림이고, 4-bit면 크기가 1/4 — 그래서 16GB 카드에 14B가 들어갑니다. 손실은 있지만 잘 만든 기법(AWQ 등)은 중요한 부분을 지켜주고, 같은 VRAM이면 큰 모델의 4-bit가 대개 이깁니다.
