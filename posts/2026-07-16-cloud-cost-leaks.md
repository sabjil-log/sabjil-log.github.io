> **한 줄 요약:** 요금이 새는 곳은 화려한 신규 서비스가 아니라 **잊혀진 것들**입니다 — 떠돌이 IP·디스크, 쌓이기만 하는 스냅샷, 새벽에도 도는 개발 서버, 안 줄어드는 오토스케일링, 그리고 조용히 나가는 전송량. **수도꼭지가 아니라 배관 누수**를 찾아야 해요.

## 왜 항상 "모르는 새" 새나

클라우드 요금은 **시간 단위 종량제**입니다. 수도요금과 같아서, 쓰는 건 눈에 보이는데 **잠그는 걸 잊은 것**은 안 보여요. 프로젝트가 끝나도 자원은 스스로 사라지지 않습니다 — 삭제 버튼을 누른 것만 사라지죠. 그래서 비용 점검은 "뭘 쓰고 있나"가 아니라 **"뭘 쓰고 있는지도 몰랐나"**를 찾는 일입니다.

## 누수 ① — 고아 자원: 주인 잃은 IP와 디스크

서버는 지웠는데 딸려 있던 **공인 IP와 블록 스토리지는 남는** 경우가 많습니다. 특히 공인 IP는 "연결 안 된 상태"일 때 오히려 과금하는 벤더가 대부분이라, 지운 서버의 IP가 매달 조용히 청구됩니다.

```bash
# AWS 예시 — 어디에도 안 붙은 EIP / 볼륨 색출
aws ec2 describe-addresses --query 'Addresses[?AssociationId==null].PublicIp'
aws ec2 describe-volumes --filters Name=status,Values=available \
  --query 'Volumes[].[VolumeId,Size,CreateTime]' --output table
```

NCP도 콘솔에서 "미사용 공인 IP", "연결되지 않은 스토리지" 필터로 같은 점검이 됩니다. **서버 삭제 = 서버+IP+디스크 삭제**를 팀 규칙으로 박으세요.

## 누수 ② — 무한 증식하는 스냅샷·백업

"매일 백업"을 걸어두고 **보존 기간을 안 정한** 경우. 스냅샷은 하나하나는 싸 보여서 방심하는데, 365개가 쌓이면 원본보다 비쌉니다. 자동 백업에는 반드시 **보존 정책(예: 일 7개 + 주 4개 + 월 3개)**을 짝지어 주세요. 지금 쌓인 개수부터 세보면 대부분 놀랍니다.

## 누수 ③ — 새벽에도 도는 개발·테스트 서버

개발 서버는 하루 8~10시간만 필요한데 24시간 돌면 **요금의 60% 이상이 아무도 없는 시간**에 나갑니다. 평일 저녁 자동 정지 + 아침 자동 시작 스케줄만 걸어도 개발 환경 비용이 뚝 떨어져요. "끄면 다시 켜기 귀찮아서"가 이유라면, 그 귀찮음을 스케줄러(크론 한 줄, 또는 벤더의 인스턴스 스케줄 기능)에게 넘기면 됩니다. 단, 정지해도 디스크·IP 요금은 남으니 ①과 세트로 관리해야 하고요.

## 누수 ④ — 늘기만 하는 오토스케일링과 과체중 인스턴스

한 번 늘어난 서버가 안 줄어드는 문제는 전에 자세히 다뤘으니(축소 정책·쿨다운·min 값) 요점만: **스케일링 이력을 열어 축소가 실제로 일어나는지** 확인하세요. 여기에 하나 더 — **애초에 큰 옷을 입은 서버**들. CPU 평균 5%로 몇 달째인 서버는 오토스케일링 이전에 사이즈 자체를 줄여야 합니다. 모니터링에서 "평균 CPU 10% 미만, 30일" 필터 한 번이면 후보가 줄줄이 나와요.

## 누수 ⑤ — 보이지 않는 전송량 (특히 밖으로, 그리고 층 사이로)

전송량 과금의 원칙: **들어오는 건 무료, 나가는 건 유료, 같은 클라우드 안에서도 존/리전을 건너면 유료**일 수 있다. 단골 사고 둘 —

- **오리진 직접 노출:** 이미지·영상을 CDN 없이 서버/버킷에서 직접 서빙 → 전송량 폭탄. CDN을 앞에 세우면 성능과 비용이 동시에 잡힙니다 (지난 CDN 글의 구조 그대로).
- **수다스러운 내부 통신:** AZ를 건너다니는 대량의 서비스 간 트래픽, 외부 API로 나가는 대용량 로그 전송. 아키텍처 그림에 **"이 화살표는 유료인가?"**를 한 번씩 물어보세요.

## 습관이 반이다

- **태그 규칙:** 모든 자원에 `owner`, `project`, `expire` 태그. 태그 없는 자원 = 삭제 후보라는 문화가 고아 자원을 원천 차단합니다.
- **예산 알람:** 월 예산의 50/80/100%에서 알림. 고지서가 아니라 **알람으로** 먼저 아는 게 목표입니다.
- **월 1회 30분:** 위 다섯 항목을 도는 정기 점검. 처음 한 번이 제일 많이 잡히고, 그다음부턴 유지비 수준입니다.

## 한 줄 정리

클라우드 비용 절감의 8할은 신기술이 아니라 **청소**입니다. 고아 IP·디스크, 무기한 스냅샷, 밤샘 개발 서버, 안 줄어드는 스케일링, CDN 없는 전송 — 이 다섯 군데만 월 1회 돌아도 고지서가 달라져요.
