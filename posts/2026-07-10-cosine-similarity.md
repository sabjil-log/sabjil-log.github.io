> **한 줄 요약:** 임베딩끼리 비슷한지 잴 때는 **길이(크기)를 무시하고 방향(각도)만** 봅니다. 두 벡터가 **같은 쪽을 가리키면** 뜻이 비슷하다고 보는 것 — 이게 코사인 유사도예요.

## 배경

RAG나 검색에서 문장을 임베딩(숫자 벡터)으로 바꾼 뒤, "질문과 가장 비슷한 문장"을 찾습니다. 그런데 '비슷함'을 어떻게 숫자로 잴까요? 직선 거리로 잴 것 같지만, 실제로는 대부분 **코사인 유사도(방향의 닮음)**를 씁니다. 왜일까요?

## 비유: 나침반 방향 vs 걸은 거리

두 사람이 각자 목적지를 가리킨다고 해봅시다.

- **직선 거리**로 비교하면, 한 명이 팔을 길게 뻗고 한 명이 짧게 뻗은 것만으로도 "다르다"고 나옵니다. 하지만 **가리키는 방향이 같다면** 두 사람 뜻은 같죠.
- **코사인 유사도**는 팔 길이(벡터 크기)는 무시하고 **가리키는 방향(각도)**만 봅니다. 방향이 같으면 1, 직각이면 0, 정반대면 -1.

문장 임베딩에서 **벡터의 크기는 대략 "문장이 얼마나 기냐/강하냐"** 같은 것에 휘둘리기 쉽고, **방향이 "무슨 뜻이냐"**를 담습니다. 그래서 뜻의 닮음을 보려면 방향, 즉 각도가 맞아요.

## 수식은 딱 이만큼

두 벡터의 코사인 유사도는 "내적을 각자 길이로 나눈 값"입니다. 길이로 나누는 그 과정이 곧 **크기 무시**예요.

```
cos(θ) = (A · B) / (|A| × |B|)
```

- `1` 에 가까움 → 같은 방향(뜻이 비슷)
- `0` → 무관
- `-1` → 정반대

## 코드로 보면

```python
import numpy as np

def cosine(a, b):
    a, b = np.array(a), np.array(b)
    return a @ b / (np.linalg.norm(a) * np.linalg.norm(b))

dog  = embed("강아지가 공원에서 뛴다")
puppy= embed("반려견이 산책 중이다")
fx   = embed("오늘 환율이 급등했다")

print(cosine(dog, puppy))   # 높게 (예: 0.8+)
print(cosine(dog, fx))      # 낮게 (예: 0.1)
```

## 실무 팁

- 많은 임베딩 모델은 벡터를 **미리 정규화(길이 1)** 해서 내보냅니다. 이 경우 내적만 계산해도 코사인과 같아져서 **검색이 더 빠릅니다**.
- 벡터 DB에서 거리 지표를 고를 때 `cosine` 또는 (정규화돼 있다면) `dot product` 를 선택하면 됩니다. 정규화 안 된 벡터에 유클리드 거리를 그냥 쓰면 문장 길이에 휘둘릴 수 있어요.

## 한 줄 정리

의미의 닮음은 **얼마나 멀리 있냐가 아니라 어느 쪽을 가리키냐**로 잽니다. 그래서 임베딩 검색의 기본은 코사인 유사도 — 크기는 버리고 방향만 봅니다.
