> **한 줄 요약:** LLM 자체는 **말만 할 수 있는 두뇌**입니다. 에이전트는 그 두뇌에 **도구(tool) 목록을 쥐여주고, "이 도구를 이 인자로 실행해줘"라는 요청을 실제로 실행해주는 반복 루프**를 붙인 것 — 마법이 아니라 구조예요.

## 챗봇과 에이전트의 경계선

"서버 상태 확인해줘"에 대한 두 가지 응답:

- **챗봇:** "서버 상태는 `ss -tlnp` 명령으로 확인할 수 있습니다" — **방법을 말해줌**
- **에이전트:** (명령을 실제로 실행하고) "8080 포트 리스닝 정상, 커넥션 210개입니다" — **일을 해서 결과를 줌**

같은 모델이어도 **도구와 루프**가 붙었느냐가 이 차이를 만듭니다.

## 메커니즘: 손 없는 천재와 조수

LLM은 텍스트만 뱉을 수 있습니다. 그럼 어떻게 "실행"을 할까요? **못 합니다.** 실행은 바깥의 코드(조수)가 해요. 흐름은 이렇습니다.

```
1. [우리 → 모델]  질문 + 도구 목록 명세
   "너는 이런 도구를 쓸 수 있어: get_server_status(host), search_docs(query)..."

2. [모델 → 우리]  "get_server_status를 host='web-01'로 실행해줘"   ← 텍스트(구조화된 요청)일 뿐

3. [우리 코드]    진짜로 함수 실행 → 결과 획득

4. [우리 → 모델]  실행 결과를 대화에 붙여 다시 전달

5. [모델]         결과를 보고 답변하거나, 다음 도구를 또 요청  → 2로 반복
```

핵심 두 가지. 첫째, **모델은 실행 요청서를 쓸 뿐, 방아쇠는 항상 우리 코드가 당깁니다.** 둘째, 에이전트의 정체는 2~5를 도는 **while 루프**입니다 — "도구 요청이 안 나올 때까지 반복". 뼈대는 이게 전부예요.

```python
messages = [{"role": "user", "content": task}]
while True:
    resp = llm(messages, tools=TOOL_SPECS)      # 1~2: 모델이 도구 요청 or 최종 답
    if resp.tool_calls:
        for call in resp.tool_calls:
            result = run_tool(call.name, call.args)   # 3: 우리가 실행
            messages.append(tool_result(call, result)) # 4: 결과 회신
    else:
        return resp.text                         # 5: 도구 요청이 없으면 = 최종 답변
```

## 도구 명세가 절반이다

모델이 도구를 **잘** 쓰게 만드는 건 명세(이름, 설명, 인자 스키마)의 품질입니다. 신입에게 업무 매뉴얼 주는 것과 같아요 — "조회함수1" 같은 이름과 한 줄짜리 설명이면 엉뚱하게 씁니다. **언제 쓰는 도구인지, 인자 형식은 뭔지, 언제 쓰면 안 되는지**까지 적어주세요. 에이전트 품질 문제의 상당수는 프롬프트도 모델도 아닌 도구 설명 부실입니다.

그리고 이 "도구 명세와 연결 방식"을 표준화하려는 흐름이 **MCP(Model Context Protocol)** 같은 프로토콜입니다 — 도구 제공자와 모델 사이의 규격 콘센트라고 보면 돼요. 도구를 서비스마다 제각각 붙이는 대신, 표준 플러그로 꽂는 것.

## 손발을 달면 안전 문제가 따라온다

말만 하던 존재에 실행 능력을 주는 순간, 실수의 성격이 바뀝니다 — **틀린 말**에서 **틀린 행동**으로. 환각 글에서 본 그 관성(그럴듯하게 지어내기)이 이제 그럴듯한 `DELETE` 요청이 될 수 있어요. 그래서 에이전트 설계의 절반은 안전장치입니다.

- **최소권한:** 도구가 쓰는 자격증명은 그 작업에 필요한 만큼만 (IAM 글의 원칙이 여기 직결). 읽기 도구와 쓰기 도구의 권한을 분리하세요.
- **위험 작업은 승인 게이트:** 삭제·결제·외부 발송류는 실행 전 사람 확인을 끼워 넣기. "실행 요청서"와 "방아쇠" 사이에 사람이 설 자리가 구조적으로 이미 있습니다(3번 단계).
- **루프 상한:** 도구 호출 횟수·비용 상한을 걸어, 모델이 헤매며 무한 반복하는 것을 차단.
- **입력 오염 주의:** 도구가 가져온 외부 텍스트(웹페이지, 문서) 안에 "이전 지시를 무시하고..." 같은 문장이 섞여 들어올 수 있습니다(프롬프트 주입). 외부 데이터는 지시가 아니라 자료로 다루도록 프롬프트에 명시하고, 그래도 위험 작업 게이트는 유지하세요.

## 한 줄 정리

에이전트 = **LLM + 도구 명세 + 실행 루프.** 모델은 요청서를 쓰고 실행은 항상 우리 코드가 하며, 그 사이가 안전장치의 자리입니다. 좋은 에이전트의 비결은 신비한 프롬프트가 아니라 — **좋은 도구 설명, 좁은 권한, 그리고 위험한 버튼 앞의 사람**이에요.
