> **한 줄 요약:** **ALB는 편지를 뜯어 내용(URL, 헤더)을 읽고** 분배하는 L7 장비, **NLB는 봉투 겉면(IP:포트)만 보고** 초고속 분배하는 L4 장비입니다. HTTP 서비스의 똑똑한 라우팅은 ALB, 비HTTP·초고성능·고정IP가 필요하면 NLB — 기준은 이게 전부예요.

## 같은 "로드밸런서"인데 층이 다르다

우편 분류소로 비유하면 —

- **NLB (L4):** 봉투 겉면의 주소·번호만 보고 던지는 분류기. **내용은 안 봅니다.** 그래서 무식하게 빠르고, 내용물이 뭐든(HTTP든 DB 프로토콜이든 게임 패킷이든) 상관없어요.
- **ALB (L7):** 편지를 **뜯어서 읽는** 분류 담당자. "청구서(/api)는 3번 창구로, 민원(/admin)은 별관으로" 같은 **내용 기반 분기**가 됩니다. 대신 뜯어 읽는 만큼 느리고, HTTP(S)라는 규격 편지만 다룹니다.

이 한 문장의 차이에서 아래 모든 실무 차이가 파생됩니다.

## ALB가 되는 것들 (내용을 읽으니까)

```
/api/*      → api 서버 그룹
/static/*   → 정적 파일 서버 그룹
호스트가 admin.example.com → 관리자 그룹
헤더에 X-Beta: true → 베타 서버 그룹 (카나리 배포)
```

- **경로/호스트/헤더 라우팅** — 마이크로서비스 여러 개를 LB 하나 뒤에 세우는 표준 패턴
- **TLS 종료 + 인증서 관리** — HTTPS를 LB에서 풀고 뒤는 HTTP로
- **리다이렉트/고정응답** — HTTP→HTTPS 리다이렉트를 서버 없이 LB가 처리
- **HTTP 헬스체크** — 경로·상태코드 기반 (그 헬스체크 3대장 글의 무대가 주로 여기)

## NLB가 되는 것들 (안 읽으니까)

- **비HTTP 프로토콜** — DB, MQTT, 게임서버, SMTP… TCP/UDP면 뭐든
- **초고성능·저지연** — 패킷을 거의 통과시키는 수준이라 초당 수백만 연결급
- **고정 IP** — NLB에는 존별 고정 IP(EIP)를 붙일 수 있습니다. 이게 은근 결정타예요: 고객사 방화벽에 "우리 IP 등록해주세요" 해야 하는 B2B 연동은 ALB(IP가 변동)로는 곤란하고 NLB가 답입니다
- **클라이언트 IP 투명 전달** — 겉봉투를 안 뜯으니 발신자 주소가 보존되는 구성이 가능

## 실무 함정 — 클라이언트 IP는 어디로 갔나

ALB 뒤의 서버에서 접속 로그를 보면 전부 **ALB의 IP**입니다. 편지를 뜯어 **새 봉투에 옮겨 담아** 보내는 구조(프록시)라 발신자가 ALB로 바뀌거든요. 진짜 클라이언트 IP는 ALB가 **`X-Forwarded-For` 헤더**에 적어줍니다 — 앱/웹서버 로그 포맷에 이걸 반영 안 하면 "모든 접속자가 같은 IP"인 이상한 로그가 됩니다.

```nginx
# nginx 예: 진짜 클라이언트 IP를 로그와 앱에 복원
set_real_ip_from 10.0.0.0/16;        # ALB 대역을 신뢰
real_ip_header X-Forwarded-For;
```

NLB는 반대의 함정 — IP는 보존되는데, 그 말은 서버 보안그룹이 **클라이언트 IP 기준**으로 열려 있어야 한다는 뜻입니다. "LB 대역만 열면 되겠지"로 잡았다가 NLB 헬스체크·트래픽이 막히는 사고가 여기서 납니다. 헬스체크 3대장의 방화벽 항목을 NLB에선 이 관점으로 다시 봐야 해요.

## NCP에서도 같은 그림

NCP Load Balancer도 **Application(L7) / Network(L4)** 타입으로 나뉘고, 고르는 기준은 동일합니다. 이름 지도 글에서 말한 그대로 — 개념은 1:1, 세부 스펙(지원 프로토콜, 고정 IP 방식)만 문서로 재확인하면 됩니다.

## 고르기 — 질문 세 개

```
1. 트래픽이 HTTP(S)인가?           → 아니오: NLB로 종결
2. 경로/호스트 라우팅, TLS 종료, 리다이렉트가 필요한가? → 예: ALB
3. 고정 IP 또는 극한 성능/저지연이 필요한가?           → 예: NLB
```

둘 다 필요하면? 겹쳐 씁니다 — **NLB(고정 IP 제공) → ALB(L7 라우팅) → 서버** 체인은 B2B 연동 + 마이크로서비스 라우팅을 동시에 만족시키는 흔한 구성이에요.

## 한 줄 정리

**뜯어 읽으면 ALB, 겉만 보면 NLB.** HTTP의 똑똑한 분기·TLS는 ALB, 비HTTP·고정IP·초고성능은 NLB — 그리고 각자의 그림자(ALB의 X-Forwarded-For, NLB의 클라이언트IP 기준 방화벽)까지 챙기면 로드밸런서 선택은 끝입니다.
