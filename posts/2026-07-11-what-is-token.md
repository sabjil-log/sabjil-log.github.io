> **한 줄 요약:** LLM은 문장을 글자도 단어도 아닌 **토큰**이라는 조각으로 잘라 읽습니다. **자주 나오는 덩어리일수록 큰 조각 하나**로 처리되고, 요금·컨텍스트 한도·속도가 전부 이 조각 개수로 계산돼요.

## 왜 알아야 하나

API 요금표는 "토큰당 얼마"고, 모델 스펙은 "컨텍스트 20만 토큰"이라고 적혀 있습니다. 그런데 토큰이 뭔지 모르면 이 숫자들이 감이 안 오죠. "A4 몇 장인데?"에 답을 못 하는 상태랄까요.

## 비유: 레고 블록

토크나이저(문장을 토큰으로 자르는 도구)는 문장을 **레고 블록**으로 분해합니다. 규칙은 하나 — **자주 쓰이는 덩어리일수록 전용 블록이 있다**는 것.

- `the`, `ing`, `안녕하세요` 처럼 흔한 덩어리 → **큰 블록 하나**
- 희귀한 단어, 오타, 신조어 → 전용 블록이 없어 **작은 블록 여러 개**로 쪼갬

그래서 같은 뜻이라도 어떤 언어·표현이냐에 따라 블록 수가 달라집니다. 대략 영어는 **1토큰 ≈ 0.75단어**, 즉 100토큰이면 영어 단어 75개쯤이에요.

## 한국어는 왜 더 먹나

토크나이저 학습 데이터에 영어가 압도적으로 많다 보니, 영어엔 큰 블록이 풍부하고 한국어는 상대적으로 잘게 쪼개집니다. 같은 내용의 문서라도 **한국어가 영어보다 토큰을 1.5~2배쯤 더 쓰는** 경우가 흔해요. 같은 API 예산으로 영어 문서는 더 길게, 한국어 문서는 더 짧게 들어간다는 뜻입니다. (모델·토크나이저마다 비율은 다릅니다.)

## 직접 세어보기

감으로 때려잡지 말고 세보면 됩니다. OpenAI 계열은 `tiktoken`으로:

```python
import tiktoken

enc = tiktoken.get_encoding("cl100k_base")
for s in ["hello world", "안녕하세요 세계"]:
    ids = enc.encode(s)
    print(s, "→", len(ids), "tokens:", [enc.decode([i]) for i in ids])
# 같은 인사말인데 한국어가 더 많은 조각으로 쪼개지는 걸 볼 수 있음
```

Anthropic·기타 모델은 각자 토크나이저가 달라 개수도 다릅니다. 정확한 값은 각 API의 토큰 카운트 기능으로 확인하는 게 맞아요.

## 이 개수가 좌우하는 세 가지

1. **요금** — 입력·출력 토큰 수 × 단가. 프롬프트에 매번 딸려 보내는 장문의 지시문은 그대로 고정비가 됩니다.
2. **컨텍스트 한도** — "20만 토큰"은 모델이 **한 번에 볼 수 있는** 총량(질문+문서+답변 전부 포함). 넘치면 앞부분이 잘리거나 요청이 거부돼요.
3. **속도** — 출력이 토큰 단위로 생성되므로, 긴 답 = 오래 걸림.

그래서 실무 절약 포인트는 단순합니다. **반복 프롬프트는 짧게 다듬고, 문서는 필요한 부분만 잘라 넣고(RAG가 하는 일이 정확히 이것), 출력 길이를 명시적으로 제한**하는 것.

## 한 줄 정리

토큰은 "자주 쓰는 덩어리일수록 큰 조각"으로 자른 레고 블록이고, LLM의 **돈·기억력·속도가 전부 이 블록 개수**로 굴러갑니다. 한국어는 영어보다 블록을 더 먹는다는 것까지 기억하면 예산 감이 잡혀요.
