> **한 줄 요약:** LLM은 다음 토큰을 **확률 뽑기**로 고릅니다. **temperature는 그 확률 분포를 얼마나 평평하게 만들지**(흔들기 세기), **top-p는 뽑기통에 넣을 후보를 어디까지 허용할지**(후보 컷오프)를 정하는 다이얼이에요. 목적에 따라 돌리는 법이 다릅니다.

## 전제: LLM은 매번 뽑기를 한다

"대한민국의 수도는" 다음에 올 토큰으로 모델은 후보마다 확률을 매깁니다.

```
서울   92%
서울특  4%
바로   2%
과연   1%
...
```

여기서 그냥 1등만 뽑으면(greedy) 항상 같은 답이 나오고, 확률대로 **추첨**하면 가끔 2, 3등도 뽑혀 다양성이 생깁니다. temperature와 top-p는 이 **추첨 방식**을 조절하는 두 개의 다이얼이에요.

## temperature — 뽑기통을 흔드는 세기

temperature는 확률 분포의 **뾰족함**을 조절합니다.

- **낮음 (0~0.3):** 분포가 더 뾰족해짐 → 1등이 거의 독식. 정답이 정해진 일(코드, 요약, 분류, 사실 질의)에 적합. 0에 가까우면 사실상 매번 같은 답.
- **중간 (0.7 전후):** 자연스러운 다양성. 일반 대화의 무난한 기본값.
- **높음 (1.0~):** 분포가 평평해짐 → 하위 후보도 곧잘 뽑힘. 브레인스토밍·창작에선 신선하지만, 사실 질의에선 **헛소리 확률도 같이 올라갑니다.**

비유하면 복권 추첨통을 흔드는 세기입니다. 살살 흔들면 위에 있던 공(1등 후보)이 나오고, 세게 흔들수록 바닥의 공도 올라와요.

## top-p — 뽑기통에 넣을 후보의 컷오프

top-p(nucleus sampling)는 **누적 확률이 p가 될 때까지의 상위 후보만** 추첨 대상에 넣습니다.

```
top_p = 0.9 라면:
서울(92%) → 누적 92% ≥ 90%  → 여기서 컷.
'서울' 하나만 뽑기통에 들어감

애매한 문장에서 후보가 고르게 퍼져 있다면:
후보 A(30%) + B(25%) + C(20%) + D(15%) → 누적 90% → 4개가 들어감
```

핵심은 **컷이 상황에 따라 유동적**이라는 것. 모델이 확신할 땐 후보 1~2개, 애매할 땐 여러 개 — 그래서 "확률 꼬리의 이상한 후보"만 깔끔하게 잘라냅니다. temperature를 높여 다양성을 살리더라도 top-p가 최후의 안전망으로 헛소리 후보를 걸러주는 관계예요.

## 실전 레시피

| 용도 | temperature | 비고 |
|---|---|---|
| 코드 생성, 데이터 추출, 분류 | 0 ~ 0.3 | 재현성·정확성 우선 |
| 일반 답변, 문서 작성 | 0.5 ~ 0.8 | 무난한 기본값 |
| 브레인스토밍, 창작, 카피 | 0.9 ~ 1.2 | 사실성 검증은 별도로 |

```python
# 같은 프롬프트, 다른 다이얼 — 직접 체감해보기
for t in (0.0, 0.7, 1.2):
    r = client.messages.create(
        model=MODEL, max_tokens=100, temperature=t,
        messages=[{"role": "user", "content": "신제품 텀블러 이름 5개만"}],
    )
    print(f"--- temperature={t} ---\n{r.content[0].text}\n")
# 0.0: 무난한 이름 반복 / 1.2: 기발하지만 가끔 산으로 감
```

주의 둘. **temperature와 top-p를 동시에 극단으로 조정하지 마세요** — 상호작용이 직관을 벗어나 디버깅이 어려워집니다. 보통 하나만 움직이는 걸 권장해요. 그리고 **temperature=0이어도 완전한 결정성은 보장되지 않을 수 있습니다**(구현·하드웨어에 따라 미세한 변동). "거의 같다" 정도로 기대하세요.

## 흔한 오해

- **"temperature 높으면 더 똑똑해진다"** — 아닙니다. 모델의 지식은 그대로고, **뽑기 방식**만 바뀝니다. 창의적으로 보이는 건 낮은 확률의 후보가 뽑혀서예요.
- **"환각은 temperature 0으로 막는다"** — 줄긴 하지만 못 막습니다. 모델이 애초에 틀린 확률 분포를 갖고 있으면 1등 후보 자체가 틀린 답이니까요. 환각 대책은 근거 주입(RAG)이 본질입니다.

## 한 줄 정리

temperature는 **흔드는 세기**, top-p는 **후보 컷오프**. 정답이 있는 일엔 낮게, 발상이 필요한 일엔 높게 — 그리고 다이얼은 한 번에 하나만 돌리세요.
