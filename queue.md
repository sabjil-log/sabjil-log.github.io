# 글감 큐 (queue)

새벽 배치가 위에서부터 하루 N개씩 꺼내 씁니다. 다 쓴 줄은 자동 삭제됩니다.
형식: `- 주제 설명 :: 카테고리`  (카테고리는 힌트, 생략 가능)
카테고리: 트러블슈팅 | 리눅스 | 클라우드 | 네트워크 | 보안 | AI/LLM

## 기둥1 — 리눅스 / 네트워크 / 도커 / 쿠버 / 조합 트러블슈팅
- 도커 이미지 레이어가 뭔데 캐시가 되는 건지, Dockerfile 순서가 빌드 속도를 좌우하는 이유 :: 트러블슈팅
- `top`/`htop` 숫자 읽는 법 — load average가 CPU 개수와 무슨 상관인지 :: 리눅스
- 디스크가 꽉 찼다는데 `df`로는 여유가 있다? inode 고갈 이야기 :: 트러블슈팅
- `systemd` 서비스가 자꾸 죽을 때 `journalctl`로 원인 찾는 순서 :: 리눅스
- `curl -v` 출력 한 줄씩 읽기 — DNS/TCP/TLS/HTTP 어디서 막혔는지 :: 네트워크
- `ss`로 보는 소켓 상태 — TIME_WAIT가 잔뜩 쌓이는 건 문제인가 :: 네트워크
- MTU와 패킷 조각화 — VPN 붙이면 특정 사이트만 안 열리는 이유 :: 네트워크
- 도커 컨테이너에서 `localhost`가 안 통하는 이유와 네트워크 모드 :: 트러블슈팅
- 쿠버네티스 Pod가 `CrashLoopBackOff` 일 때 읽어야 할 것들 :: 트러블슈팅
- 쿠버네티스 Service/Ingress가 뭔지 — 트래픽이 Pod까지 오는 길 :: 네트워크
- `kubectl` 필수 디버깅 명령 5개 (logs/describe/exec/get -o/port-forward) :: 리눅스
- 리눅스 파일 권한 `chmod 755`의 숫자가 진짜 뜻하는 것 :: 리눅스
- 프로세스는 죽었는데 포트가 안 풀릴 때 (`lsof -i`, TIME_WAIT) :: 트러블슈팅
- 크론이 손으로는 되는데 스케줄로는 안 도는 흔한 이유 (PATH/환경변수) :: 트러블슈팅
- `tcpdump` 기본기 — 캡처해서 "누가 응답을 안 하는지" 보기 :: 네트워크
- 도커 볼륨 vs 바인드 마운트 — 데이터가 사라지는 사고 막기 :: 트러블슈팅
- OOM Killer가 내 프로세스를 죽였다 — `dmesg`로 확인하기 :: 트러블슈팅

## 기둥2 — NCP / AWS 서비스
- NCP Object Storage에 boto3 물릴 때 checksum·서명 함정 :: 클라우드
- VPC 서브넷을 public/private로 나누는 이유와 라우팅 테이블 :: 네트워크
- NAT Gateway가 뭔지 — private 서버가 외부로 나가는 문 :: 네트워크
- 오토스케일링이 커지기만 하고 안 줄어드는 이유 (쿨다운/지표) :: 클라우드
- 클라우드 비용이 새는 곳 톱5와 확인법 :: 클라우드
- 관리형 DB로 옮길 때 놓치는 것 (파라미터그룹/백업/장애조치) :: 클라우드
- KMS 봉투암호화(envelope encryption)가 뭔지 그림으로 :: 보안
- Secret Manager를 왜 쓰나 — 코드에 비밀번호 박으면 생기는 일 :: 보안
- 클라우드 IAM 최소권한 원칙 — 읽기전용부터 시작하는 습관 :: 보안
- CDN 캐시가 안 갱신될 때 (TTL/무효화/캐시키) :: 클라우드
- 오브젝트 스토리지 정적 웹 호스팅 — S3/Object Storage로 사이트 띄우기 :: 클라우드

## 기둥3 — AI / LLM 학습
- 임베딩 거리(코사인 유사도)가 뭔지 — 왜 각도로 재나 :: AI/LLM
- 벡터 DB가 일반 DB와 뭐가 다른지 — 근사 최근접 검색(ANN) :: AI/LLM
- 청크 크기가 RAG 품질을 좌우하는 이유와 오버랩 :: AI/LLM
- 리랭커(reranker)는 왜 필요한가 — 1차 검색의 한계 :: AI/LLM
- 컨텍스트 윈도우가 꽉 차면 생기는 일과 대처 :: AI/LLM
- 온도(temperature)와 top-p — 창의성 다이얼 읽는 법 :: AI/LLM
- 양자화(4-bit)가 뭐길래 16GB VRAM에 14B 모델이 올라가나 :: AI/LLM
- 프롬프트에 "근거만 써라"를 넣어야 환각이 주는 이유 :: AI/LLM
- LLM이 최신 정보를 모르는 이유(지식 컷오프)와 해결책 :: AI/LLM
